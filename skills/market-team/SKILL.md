# /market-team

作ったものをどう届けるか。3つのマーケティング視点からサブエージェントで並列調査し、結果を統合してGo-to-Market戦略の材料を報告する。実装はしない。

## 調査員（サブエージェント — 使い捨て）

| 調査員 | 視点 | subagent_type |
|--------|------|---------------|
| **A** | ポジショニング・メッセージング（CORE.md逆算、訴求軸導出） | Explore |
| **B** | チャネル・成長戦略（WebSearchで配信経路調査） | general-purpose |
| **C** | 競合マーケティング分析（競合の訴求・LP・集客手法） | general-purpose |

## ルール

1. **調査だけ。** 実装・コード変更はしない。
2. **届く実感で優先度をつけろ。** 判定基準: その施策を実行したらユーザーに届く量・質が変わるか？
3. **統合は「整理」ではなく「批判的思考」。** 3視点の結果をきれいに並べるだけでは不十分。調査員間の矛盾を洗い出し、楽観バイアスを疑い、「で、本当はどうすべきか」まで踏み込む。
4. **聞くのは1回。** プロダクト概要とターゲットの確認だけ。指定なしならCORE.md/SPEC.mdから推定。

## フロー

1. ユーザーにプロダクト概要・ターゲット・調査テーマを確認（CORE.md/SPEC.mdがあればそこから推定し、確認のみ）
2. docs/CORE.md, docs/SPEC.md があれば読む
3. A/B/C を Task で**並列起動**（1つのメッセージで3つ同時）
4. 結果を統合して報告（実行インパクト順）
5. **矛盾分析（Devil's Advocate）** — 統合結果に対して以下を実行:
   - 調査員間の矛盾を洗い出す（Aの推奨とBの推奨が衝突していないか？）
   - 楽観バイアスを疑う（「競合不在」は需要不在かもしれない。「バズる」は願望かもしれない）
   - プロダクトの実装と訴求メッセージの乖離を検証する（実態が伴わない訴求はリスク）
   - ターゲットとチャネルの整合性を確認する（推奨チャネルにターゲットは本当にいるか？）
   - 未検証の前提を列挙する（PMF、技術的制約、コスト構造など）
   - 矛盾を踏まえた「本当にやるべきこと」を優先度付きで提示する
6. 統合レポート + 矛盾分析を `docs/current/` に保存
   - ファイル名: `YYYY-MM-DD_HHmm_market_research.md`（実行時刻）
   - フォルダがなければ `mkdir -p docs/current` で作成
   - 保存したパスをユーザーに報告

## 調査員への指示

### A（ポジショニング・メッセージング調査）
- CORE.md/SPEC.mdを読み、プロダクトの本質的価値を抽出
- 観点：
  - ユーザーが得る核心的な便益（機能ではなく体験・変化）
  - 一言で伝わるバリュープロポジション案（3パターン以上）
  - ターゲットユーザー像（誰の・どんな課題を・どう解決するか）
  - 訴求の切り口（機能訴求 / 感情訴求 / 社会的証明 / 希少性 等）
  - LP・紹介文で使えるキャッチコピー案
- 重要：「機能の羅列」ではなく「ユーザーの言葉で語れる価値」を優先
- **矛盾・リスクも報告**: 訴求メッセージとプロダクト実態の乖離、ターゲットに刺さらない可能性、自分の提案への反論も含める
- 報告フォーマット：発見（プロダクトの強みは何か）→ 訴求案 → 推奨メッセージング → **リスク・反論**
- 対象テーマ：{theme}

### B（チャネル・成長戦略調査）
- WebSearchでプロダクトカテゴリに適した配信・集客チャネルを調査
- 観点：
  - 有効なチャネル候補（SEO / SNS / コミュニティ / プロダクトハント / 広告 / メール / パートナーシップ 等）
  - 各チャネルの初期コスト・リーチ・相性
  - 同カテゴリで成功したローンチ事例
  - 初期トラクションの獲得方法（最初の100ユーザーをどう集めるか）
  - 無料 vs 有料チャネルの使い分け
- 重要：「このプロダクトに合うチャネル」を具体的に。一般論ではなくカテゴリ特化で
- **矛盾・リスクも報告**: 推奨チャネルにターゲットが本当にいるか疑え。「使えそう」だけでなく「使えない理由」も検証せよ
- 報告フォーマット：チャネル候補 → 相性評価 → 推奨アクション → **リスク・反論**
- 対象テーマ：{theme}、プロダクト概要：{product_summary}

### C（競合マーケティング分析）
- WebSearchで競合・類似サービスのマーケティング手法を調査
- 観点：
  - 競合がどんな訴求をしているか（LP・広告コピー・SNS投稿）
  - 価格戦略・プラン設計（フリーミアム / 有料 / 従量課金 等）
  - 集客チャネル（どこから流入を取っているか）
  - 差別化ポイント（競合が訴求していない空白地帯）
  - ユーザーレビュー・不満点（奪えるポイント）
- 重要：「競合がやっていること」だけでなく「競合がやっていないこと（=チャンス）」も探せ
- **矛盾・リスクも報告**: 「空白地帯」が需要不在の可能性も検証せよ。競合が強い理由・なぜそのポジションを取っているかも分析せよ
- 報告フォーマット：競合名 → 訴求・チャネル・価格 → 空白地帯 → 示唆 → **リスク・反論**
- 対象テーマ：{theme}、プロダクト概要：{product_summary}
