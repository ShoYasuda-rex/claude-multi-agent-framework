# /debate

任意のテーマを2フェーズで深掘り議論する。

| フェーズ | 性質 | 動作モード | 発言形式 |
|----------|------|-----------|----------|
| **発散** | 広げる | 3体常駐・並列・自由発言 | 2〜3行の短文のみ |
| **収束** | まとめる | 司会が1体ずつ指名 | 構造化OK |

---

## アーキテクチャ

**2チャット構成。フェーズで動作モードが変わる。**

| チャット | 役割 | 動作 |
|----------|------|------|
| **Chat 1（ホスト）** | 進行管理 | 発散: 3体を常駐で並列起動 → 収束: 司会モードで1体ずつ指名 |
| **Chat 2（ユーザー）** | ユーザーが発言 | `/debate user` で入室。発言をファイルに書き込む。閲覧はファイルを直接開く |

```
Chat 1: テーマ入力 → ファイル生成 → 発散（3体常駐）→ 収束（司会）→ 結論表示
Chat 2: /debate user → 入室 → 発言書き込み
```

---

## フロー

### 起動時（引数で分岐）

- `/debate` → 新規作成フロー（ホスト）
- `/debate user` → ユーザー参加フロー

---

## 新規作成（ホストチャット）

### ステップ1: テーマ入力

1. ユーザーにテーマを聞く
   - ユーザーが「レポートから」等と指示した場合のみ → `.reports/*_research.md` を検索・一覧表示。選択されたレポートの全文を `## 参考資料` に埋め込む（要約しない）
   - ユーザーがテーマと共に参考情報を提供した場合も、原文をそのまま `## 参考資料` に記載する（要約しない）

### ステップ2: ファイル生成

1. `.reports/` フォルダを作成（なければ）
2. `.reports/{YYYYMMDD}_{HHmm}_{theme}_debate.md` を作成（themeはテーマを短縮した英語スラッグ）
3. ヘッダー（テーマ: 確定済み・参加者: なし・フェーズ: 発散）+ 参考資料（あれば）+ `## 発散` を書き込む
4. AI 3体（`Claude-A` / `Claude-B` / `Claude-C`）を参加者リストに追記
5. 「別チャットで `/debate user` してください」と案内

### ステップ3: 発散フェーズ（3体常駐）

3体のエージェントを **並列に** 常駐起動する（Agent tool × 3、同時実行）。

各エージェントに渡す情報:
- 自分のID
- debateファイルのパス
- 発散フェーズのルール（短文のみ、自由発言）
- **「Userが参加するまで待機。参加後は自律的にファイルを読み書き。`## 収束` または `## 結論` を検知したら終了」** という指示

```
各エージェントの動作:
  ファイルを読む → User未参加なら待機
  ループ {
    ファイルを読む
    直前の発言者が自分以外 → 1発言を追記
    `## 収束` or `## 結論` 検知 → 終了
    少し待つ
  }
```

ホストは3体の終了を待つ。全員終了 = 発散フェーズ完了。

### ステップ4: 収束フェーズ（司会モード）

ホストが司会として1体ずつ指名する。ファイル末尾だけ読んで進行管理。

```
ループ {
  ファイル末尾を読む
  設計者 → 探索者 → 批評者 の順にエージェントを呼ぶ（1発言のみ）
  ユーザー発言があれば次の設計者から再開
  結論セクション検知 → 結論を表示して終了
}
```

**ホストは全文を読まない。** 末尾の変化だけ見て判断する。

各エージェントを呼ぶ際に渡す情報:
- 自分のID・役割（設計者/探索者/批評者）
- debateファイルのパス
- 収束フェーズのルール
- **「ファイルを自分で読み、1発言だけ書いて終了」** という指示

エージェントは自分でファイル全文を読み、1発言を追記して終了する。

### ステップ5: 結論表示

結論セクションを読んでチャットに表示して終了。

---

## ユーザー参加（別チャット）

### ステップ1: 入室

1. `.reports/*_debate.md` の最新ファイルを使用
2. 参加者リストに `User` を追記

### ステップ2: 発言ループ

```
ループ {
  ユーザーの発言を待つ（AskUserQuestion）
  発言をファイルに `### [User]` として追記
  結論セクション検知 → 終了
}
```

**ユーザーはいつでも発言できる。** 順番の制約なし。議論の閲覧はファイルを直接開く。

---

## Phase 1: 発散

- **役割なし。** 全員フラット。IDだけで識別。
- **2〜3行の短文のみ。** 箇条書き・見出し・構造化は禁止。
- **否定OK、賛同OK、脱線OK。** 深く考えすぎない。即書く。
- **直前の発言者以外が応答。** 連投防止。

発言: `### [{ID}]` + 2〜3行の短文

### 発散 → 収束への移行

**ユーザーの指示でのみ移行する。** AIからの収束提案は禁止。

- ユーザーが「収束」等を発言 → ホストがファイルに `## 収束` を書き込む → 常駐エージェントが検知して終了 → ホストが発散中の発言傾向から設計者/探索者/批評者を割り振り（具体案→設計者、別角度→探索者、リスク指摘→批評者）
- ユーザーが「終了」→ ホストがファイルに `## 結論` を書き込む → 常駐エージェントが検知して終了 → 設計者を1回呼んで発散の内容で結論ドキュメントを作成

---

## Phase 2: 収束（設計者 × 探索者 × 批評者）

| 役割 | やること | やらないこと |
|------|---------|-------------|
| **設計者** | 素材を整理・構造化・設計に落とす | 問題点を無視 |
| **探索者** | 新しい切り口・見落とし・可能性を投げ込む | 設計者を否定（批評者の仕事） |
| **批評者** | 穴を突く・エッジケース・実現可能性を問う | 対案を出す（設計者の仕事） |

順序: `設計者 → 探索者 → 批評者` を1ラウンドとする。

### Round 1
- **設計者**: 自分の案ではなく発散の素材整理から。対立意見も両方提示し、方針を提案
- **探索者**: 発散で拾われなかった素材・新しい角度を投げ込む
- **批評者**: 穴を突く。観点はテーマに応じて自分で設計

### Round 2〜
- **設計者**: 探索者案の採否（理由付き）+ 批評者指摘への修正or反論を全て回答
- **探索者**: さらに新しい角度を投げ込む
- **批評者**: さらに深い問題を提起

---

## 終了判定

**ユーザーの指示でのみ終了する。** AIからの終了提案は禁止。

- ユーザーが「終了」「まとめて」等を発言 → 設計者が結論ドキュメントをファイル末尾に書く
- AIは何ラウンドでもユーザーが止めるまで続ける

### ユーザー発言ルール
- **ユーザー発言の直後に結論を書いてはならない。** ユーザーの指摘は必ず設計者→探索者→批評者の1ラウンドを回してから反映する。
- ユーザー発言は「新しい論点の追加」として扱い、結論への吸収ではなく議論の継続トリガーとする。

## ID

AI: `Claude-A` / `Claude-B` / `Claude-C` — ユーザー: `User`

## 結論ドキュメント

設計者がファイル末尾に書く（別ファイルは作らない）。セクション名はテーマに応じて選ぶ。

**必須:** 概要 / 決定事項 / リスクと対策
**任意:** 採用しなかったアイデア / 技術スタック / UI設計 / データ構造 / フォルダ構成 / KPI / タイムライン等

---

## ルール

1. **ファイルが真実。** 議論の閲覧はファイルを直接開く。
2. **日本語で議論。**
3. **設計者は全指摘に回答する。** 逃げない。
4. **設計者だけが結論を書く。**
5. **ホストは議論に参加しない。** 発散では常駐管理、収束では司会に徹する。
6. **フェーズ移行・終了は全てユーザー指示のみ。** AIからの収束提案・終了提案は禁止。
7. **発散: エージェントは常駐で自律動作。** 収束: エージェントは1発言で終了、ホストが次を呼ぶ。
