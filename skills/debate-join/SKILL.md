# /debate-join

進行中の debate に参加する。引数なしでAIが1発言、`user`でユーザー常駐。

---

## フロー

### ステップ1: debateファイルを見つける・モード判別

1. `../reports/{プロジェクト名}/*_debate.md` を検索し、最新ファイルを使用（プロジェクト名 = カレントディレクトリのフォルダ名）
   - 注意: Globツールはワークスペース内限定のため、親ディレクトリ検索時はShellコマンド(`dir`/`ls`)を併用する
2. 引数で分岐:
   - `user` → **ユーザーモード**: ヘッダー（参加者リスト）のみ読む
   - 引数なし → **AIモード**: ファイル全文を読む

---

## ユーザーモード（常駐）

1. 参加者リストに `User` を追記（未参加の場合のみ）
2. 常駐ループに入る

```
ループ {
  ユーザー入力を待つ
  「離脱」「抜ける」等 → 終了
  それ以外 → `### [User]` の形式でファイルに追記 → ループ継続
}
```

---

## AIモード

### 1. 入室

参加者リストに自分のモデル名を追記する（例: `Claude-4-Opus`, `GPT-4o` 等、実際のモデル名を使用）。

### 2. フェーズ判定

ファイルの内容から現在のフェーズを判定し、発言スタイルを決める。

- `## 収束` がない → 発散フェーズ（短文）
- `## 収束` がある → 収束フェーズ（自由）

### 3. 1発言を追記

`### [自分のモデル名]` の形式でファイル末尾に1発言だけ書く。

**発散フェーズ:**
- 2〜3行の短文のみ。箇条書き・見出し・構造化は禁止。
- 既存の発言を踏まえつつ、新しい視点を投げる。

**収束フェーズ:**
- 役割なし。議論の流れに合わせて自由に発言。
- 設計案への補強、見落としの指摘、別の切り口など、場に必要なことを1発言。

### 4. 終了

発言後、書いた内容をチャットに表示して **終了**。

---

## ルール

1. **ユーザーモードは常駐。** チャットが debate への窓口になる。「離脱」「抜ける」等で退出。**AIモードは単発。**
2. **役割を持たない。** 設計者/探索者/批評者の割り当てはホスト（debate スキル）の仕事。
3. **フェーズのルールに従う。** 発散なら短文、収束なら流れに合わせる。
4. **結論は書かない。** 結論を書くのは設計者（debate スキル側）の仕事。
5. **日本語で発言。**
