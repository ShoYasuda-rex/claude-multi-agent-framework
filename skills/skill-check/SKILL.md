# /skill-check

スキル実行後に同じチャットで呼ぶと、直前のスキルの定義と実行結果を比較して差異をレポートする。

---

## フロー

### ステップ1: 対象スキルの特定

1. 引数にスキル名がある → そのスキルを対象にする（例: `/skill-check debate`）
2. 引数なし → チャット履歴から直前に実行されたスキルを自動特定する
3. `skills/{name}/SKILL.md` を読み込む

### ステップ2: チェック項目の抽出

SKILL.md を読んで以下を抽出する:

- **成果物**: 作成・更新されるべきファイル（パスパターン、必須セクション）
- **フロー**: 実行されるべきステップの順序
- **ルール**: 禁止事項・必須事項
- **ツール使用**: 使われるべきツール（Agent, Read, Write 等）

### ステップ3: 実行結果の収集

**チャット履歴から:**
- 呼ばれたツールとその結果
- 作成・更新されたファイルパス
- Agent（サブエージェント）の呼び出しと結果
- エラーや中断の有無

**ファイルから:**
- Glob で成果物ファイルを特定
- Read で実際の内容を読む（必須セクションの有無、構造の確認）

### ステップ4: 突き合わせ評価

各チェック項目に対して判定する:

- **PASS**: 定義どおりに実行された
- **FAIL**: 定義に反している（必須セクション欠落、禁止事項の違反、未実行ステップ等）
- **WARN**: 確認困難だが懸念あり（時間的振る舞い、内部ループ状態、エージェントの判断品質等）

### ステップ5: レポート出力

```
## /skill-check 結果

対象スキル: {スキル名}

### 成果物チェック
- [PASS/FAIL/WARN] {項目}
- ...

### フローチェック
- [PASS/FAIL/WARN] {項目}
- ...

### ルールチェック
- [PASS/FAIL/WARN] {項目}
- ...

### 総合: PASS X / FAIL X / WARN X
```

FAIL が 0 件の場合: 「定義どおりに実行されました」とまとめる。
FAIL がある場合: 各 FAIL の原因を1行で補足する。

---

## ルール

1. **読み取り専用。** ファイルを変更しない。
2. **チェック項目は SKILL.md から動的に抽出。** スキルごとのハードコードなし。
3. **確認できない項目は WARN にする。** 推測で PASS/FAIL を出さない。
4. **日本語でレポート出力。**
