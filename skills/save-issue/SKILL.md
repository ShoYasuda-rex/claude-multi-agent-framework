# Issue（即記録）

ユーザーに内容を聞いて、issue ファイルとして即記録する。最速で終了。

---

## 手順

### 1. 内容を聞く

- 「記録する内容を入力してください:」とだけ表示して **停止する**（ユーザーの次の入力を待つ）
- ツールは使わず、テキスト出力のみ

### 2. ディレクトリ準備

- `.claude/issues/` ディレクトリがなければ Bash で作成する

### 3. 日時取得

- Bash で現在日時を取得する:
  - `date '+%Y-%m-%d %H:%M'` → 登録日用
  - `date '+%Y-%m-%d_%H%M'` → ファイル名プレフィックス用

### 4. ファイル名生成

- 入力内容から2-3語のキーワードを抽出し、kebab-case に変換する
  - 日本語の場合はローマ字化せず、意味のある英単語に要約する
  - 例: 「step3のヒント文言が硬い」→ `step3-hint-tone`
- ファイル名: `{YYYY-MM-DD}_{HHmm}_{keyword}.md`

### 5. ファイル作成

以下の内容で Write する:

```markdown
# {入力内容そのまま}
- **登録日:** {YYYY-MM-DD HH:mm}
- **ステータス:** 未対応
```

### 6. 完了報告

- 「記録しました: `{ファイル名}`」と1行だけ表示して **終了**
- 余計な説明やフォローは不要。会話を止めない。

---

## 禁止事項

- 優先度・カテゴリなどの追加情報を聞かない
- 記録後に「他にありますか？」等のフォローをしない
