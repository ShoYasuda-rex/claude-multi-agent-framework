# biz-chate-meeting DESIGN

スキル設計のフロー・参照・作成ファイルを整理したドキュメント。

---

## セクション1: フロー図詳細（エージェント担当含む）

### 発動〜モード分岐

```
[人間] 「社員ミーティングの準備」「biz-chate-meeting」等で発動
    |
    v
[Agent] 発動文言を解釈
    |
    +-- 新規生成系の文言 --→ モード1
    |
    +-- 「この内容で確定して」「議事録を更新して」等 --→ モード2
```

### モード1: 新規 .md 生成

```
[Agent] 日付の確定
    |-- ユーザー指定あり → その日付を使用
    |-- 未指定 → 「今週のミーティング日はいつですか？」と聞く [人間]
    |            または今日の日付を使用（実装依存）
    v
[Agent] Chateraise/ 内の *_社員ミーティング.md を列挙 → 最新を Read
    v
[Agent] 前回の「決定・アクション」「次回までに」を抽出 → 前回フォロー要約
    v
[Agent] references/meeting-sheet.md を Read
    v
[Agent] YYYY-MM-DD を置換、前回フォローを「前回フォロー」セクションに埋め込み
    v
[Agent] Chateraise/YYYY-MM-DD_社員ミーティング.md に Write
    v
[Agent] 完了報告（Obsidian で開く旨・「この内容で確定して」の案内）
    v
[人間] Obsidian で .md を開く
[人間] Cursor で AI と会話しながら議題を埋める
```

### モード2: 会話内容の反映

```
[Agent] 直近の会話ログから議事録に載せる内容を抽出
    v
[Agent] 対象の Chateraise/YYYY-MM-DD_社員ミーティング.md を特定
    |  ※対象の特定方法は SKILL に未記載（直近会話の日付・ファイル名等で推測）
    v
[Agent] 該当ファイルを Read
    v
[Agent] 会話で出た決定・アクション・スタッフ状況等を追記・更新
    v
[Agent] Write して報告
```

### エージェント担当一覧

| ステップ | 担当 | 備考 |
|----------|------|------|
| 発動 | 人間 | 発動文言の入力 |
| モード判定 | Agent | 発動文言からモード1/2を判定 |
| 日付確定 | Agent | 未指定時は人間に聞く or 今日を使用 |
| 前回議事録 Read | Agent | Chateraise/ 内の Glob + Read |
| 前回フォロー抽出 | Agent | テキスト要約 |
| meeting-sheet Read | Agent | references/meeting-sheet.md |
| 置換・生成 | Agent | YYYY-MM-DD、前回フォロー埋め込み |
| 新規 .md Write | Agent | Chateraise/YYYY-MM-DD_社員ミーティング.md |
| Obsidian で開く | 人間 | スキル外 |
| 会話で議題を埋める | 人間 + Agent | Cursor 内、スキル外 |
| 会話ログ抽出 | Agent | モード2 |
| 対象ファイル特定 | Agent | モード2、仕様が曖昧 |
| 更新 Write | Agent | モード2 |

### 設計上の注意点

- **モード2の対象ファイル特定**: SKILL に「対象の YYYY-MM-DD」の決め方が書かれていない。直近の会話で日付が出ている、直近の社員ミーティング.md、人間に聞く、のいずれかを明示するとよい
- **初回実行時**: Chateraise/ に *_社員ミーティング.md が存在しない場合の扱い（前回フォローを空にする等）を SKILL に追記するとよい
